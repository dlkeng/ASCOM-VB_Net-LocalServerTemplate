ASCOM Drivers and LocalServer How To

This webpage provides examples of the steps involved in setting up Visual Studio project skeleton frameworks for
development of ASCOM drivers or local servers in both Visual C# and Visual Basic using the ASCOM
Platform 6.1SP1 and the ASCOM Platform Developer Components and Driver Conformance Checker tools
on a 64-bit Windows 7 development PC.
The resulting skeleton frameworks generate "working", "non-functional" ASCOM drivers or local servers.
  - "working" means they pass the Conformance Checker tool
  - "non-functional" means they control no actual hardware (yet)

Additionally, the steps for generating a Windows installer EXE using ASCOM Driver Install Script Generator
and Inno Setup are provided.
The Driver examples demonstrate the development of the skeleton framework for an ASCOM Focuser driver
for a fictitious company/product called Acme.

The LocalServer examples demonstrate the development of the skeleton framework for local server-served ASCOM
Focuser and FilterWheel drivers for a fictitious company/product called Acme.

Each example includes a Checklist of the sequential steps involved, a Walk-Through of those steps with
annotated screen images and additional commentary, a Visual studio project of the example's code,
and a PDF file of both the Checklist and Walk-Through. (The Inno Setup example scripts are included in
the example code for its companion driver/local server.)

For Visual Basic local server development, an ASCOM LocalServer Template VB is provided.

Finally, an ASCOM drivers/local server developer FAQ, ASCOM Driver Install Script Generator information,
and a working example of a LocalServer-served Focuser & FilterWheel Drivers using a single, shared
COM port to a custom Focuser/FilterWheel controller is provided.

(The examples use the Visual Studio 2010 Express editions, but should work {untested} in the full editions
 and other newer editions with minor or no modifications.)
